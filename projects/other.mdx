## finiteCraft

finiteCraft automates the discovery of all possible crafting combinations in Neal Agarwal's browser game Infinite Craft. It uses a distributed proxy network to avoid rate limiting and implements pathfinding algorithms to efficiently find crafting recipes for any target item.

**Tech stack:** Python, Selenium, Web Scraping, Proxies, Automation

<AccordionGroup>
  <Accordion title="Features">
    - Selenium-based browser automation for game interaction
    - Proxy rotation support for distributed scraping
    - Crafting recipe database for discovered combinations
    - Pathfinding algorithms to find optimal crafting paths to a target
  </Accordion>
</AccordionGroup>

[View on GitHub](https://github.com/finiteCraft/finiteCraft)

## Molotov

Molotov is an automation bot for the browser-based word game Bomb Party on jklm.fun. It uses Selenium to interact with the game interface and an optimized word database for fast, accurate responses. JavaScript injection is used where direct DOM interaction is faster than simulated keystrokes.

**Tech stack:** Python, Selenium, JavaScript

<AccordionGroup>
  <Accordion title="Features">
    - Selenium-based browser automation for game interaction
    - Optimized word database for rapid lookup under time pressure
    - Configurable response timing to adjust bot behavior
    - JavaScript injection for performance-critical interactions
  </Accordion>
</AccordionGroup>

[View on GitHub](https://github.com/quantumbagel/Molotov)

## Chess-Challenge

Chess-Challenge is an entry for Sebastian Lague's Tiny Chess Bot Challenge, where the goal is to write the strongest possible chess AI within strict code size constraints. It implements minimax search with alpha-beta pruning and a hand-tuned evaluation function.

**Tech stack:** C#, Chess, AI

<AccordionGroup>
  <Accordion title="Features">
    - Minimax algorithm with alpha-beta pruning for move search
    - Compact code optimized to fit within the challenge's size limit
    - Custom board evaluation function tuned for competitive performance
    - Strong performance relative to the tight code size constraints
  </Accordion>
  <Accordion title="Challenge context">
    Sebastian Lague's Tiny Chess Bot Challenge restricts all entries to a fixed token budget. Every line of code counts, so the implementation favors algorithmic efficiency and hand-written evaluation heuristics over larger, table-driven approaches.
  </Accordion>
</AccordionGroup>

[View on GitHub](https://github.com/quantumbagel/Chess-Challenge)
